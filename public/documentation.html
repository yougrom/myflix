API Endpoints Documentation

The MyFlix API is designed following REST principles, featuring intuitive URLs, JSON-formatted responses, and conventional HTTP status codes. It serves as the backend for a web application that offers users detailed information on movies, directors, and genres. Additionally, it enables users to register, update their personal details, and compile a list of favorite movies.

Business Logic HTTP Method URL Request Body Format Response Body Format
Get all movies GET /movies N/A JSON list of movies
Get a single movie GET /movies/{title} N/A JSON movie details
Get genre by name GET /genres/{name} N/A JSON genre description
Get director by name GET /directors/{name} N/A JSON director details
User registration POST /users { "username": "string", "password": "string", "email": "string", "birthday": "date" } Confirmation text
Update user info PUT /users/{username} { "username": "string", "password": "string", "email": "string", "birthday": "date" } Confirmation text
Add to favorites POST /users/{username}/movies/{movieID} N/A Text indicating addition
Remove from favorites DELETE /users/{username}/movies/{movieID} N/A Text indicating removal
User deregistration DELETE /users/{username} N/A Confirmation of removal
